Для работы проекта необходим ruby 1.9.

Для ознакомления с управлением роботом советую использовать <em>script/console</em> – данный скрипт позволяет в интерактивном режиме обмениваться информацией с сервером робокапа.

Робот представлен модулем +Robot+. Чтобы посмотреть доступные методы данного модуля, напишите в консоли <tt>Robot.m</tt>.

Метод +m+ позволяет получить список всех методов (без учета стандартных) для любого объекта.

Используя <tt>reload</tt>, можно перезагрузить код проекта в консоли (полезно после редактирования кода проекта, чтобы не перезагружать консоль).

Для того, чтобы перезагрузить самого робота можно использовать <tt>Client.reconnect</tt>.

Чтобы посмотреть список созданных поз для робота, наберите <tt>Robot.pose.m</tt>. Чтобы вызвать конкретную позу, к примеру _hi_ - напишите <tt>Robot.pose.hi</tt>.

Все позы определены в формате yaml и находятся в папке <em>lib/player/pose/</em>.

Структура суставов робота:

* голова:
  <em>h1, h2</em>
  
* левая рука:
  <em>la1, la2, la3, la4</em>

* правая рука:
  <em>ra1, ra2, ra3, ra4</em>

* левая нога:
  <em>ll1, ll2, ll3, ll4, ll5, ll6</em>

* правая нога:
  <em>rl1, rl2, rl3, rl4, rl5, rl6</em>


Для получения информации о состоянии сустава робота вызовите соответствующий метод у модуля +Robot+. Например, <tt>Robot.h1</tt> – вернет информацию об угле первого сустава головы. Также можно задать угол сустава, например, <tt>Robot.h1 = 1</tt>.

Чтобы следить за состоянием определенного сустава используйте _callbacks_. <tt>throw :done</tt> - сигнализирует, что +callback+ закончил свою работу. Пример, в котором будет отображаться информация об изменении <em>h1</em>, пока величина угла не достигнет 10 градусов.

  Robot.add_callback(:h1) do |old_value, new_value|   
    puts "Угол соединения h1 изменился от #{old_value} к #{new_value}."  
    throw :done if new_value.abs > 10
  end

Когда нужно предусмотреть ситуацию с двумя конфликтующими коллбэками (к примеру, при управлении движением сустава), могут пригодиться наблюдатели (observers - по сути, именованные _callbacks_). При задании наблюдателей с одинаковым именем для одного сустава последний заданный запишется поверх первого.
